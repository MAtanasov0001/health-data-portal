# Принос към проекта

Проектът се разработва **открито** — не е добра практика, а нормативно изискване (чл. 44, ал. 1
и МЕ32 НОИИСРЕАУ). Приветстваме приноси от ИТ общността.

## Нормативен ред за приноси (чл. 60, ал. 2–4 НОИИСРЕАУ)

Лица извън администрацията и изпълнителите могат да предлагат подобрения в изходния код.
Възложителят (МИДТ) ги **разглежда в 30-дневен срок** и ги приема или **отказва мотивирано**.
Приетите предложения се добавят чрез сливане (merge). Този срок е нормативен ангажимент.

## Процес

1. Отворете issue за обсъждане, преди по-голяма промяна (освен за уязвимости — вж. `SECURITY.md`).
2. Разклонете (fork/branch), направете промяната в отделен клон.
3. Уверете се, че **всички CI порти минават** локално (вж. по-долу).
4. Отворете Pull Request и попълнете шаблона, вкл. **контролния списък за нормативно съответствие**.
5. Изчакайте ревю според режима на компонента (A/B/C — вж. `docs/review-model.md`).

## Режим на ревю (задължителен)

| Режим | Обхват | Изискване преди сливане |
|-------|--------|--------------------------|
| **A** | frontend, статични страници, визуализации, документация | автоматични порти + 1 ревюиращ |
| **B** | приемане/ETL, публично API, DCAT-AP профил, IaC, конвейер, одит | порти + ревюиращ ≠ автор + тестове с реални схеми |
| **C** | сигурност, идентичност, интеграционен слой, жив proxy, анонимизация | порти + експерт по сигурност + **двама** одобряващи; без изключения |

## Означаване на AI-генериран код

Ако принос е (частично) генериран от AI асистент, добавете ред в съобщението на комита:

```
Assisted-by: <име на инструмента/модела>
```

Служи за проследимост, калибриране на строгостта на ревюто и правна яснота (вж. концепция
раздел 13.3). Всеки принос удостоверява произхода си чрез **Developer Certificate of Origin**:

```
Signed-off-by: Име Фамилия <email>
```

## Локални проверки (същите като CI портите)

```bash
# Frontend
cd frontend && npm run lint && npm run type-check && npm test && npm run test:a11y

# Python (приемане/API)
cd ingestion && ruff check . && mypy . && pytest

# Тайни (не трябва да има нито една)
gitleaks detect --no-banner
```

## Стил и граници

- Спазвайте модулните граници на монорепото; без copy-paste между слоеве.
- Никакви тайни в кода, тестовете или примерите в документацията — **дори примерни**.
- Всяка нова потребителска функционалност има съответна публична крайна точка (API-first, чл. 41).
- Времеви стойности — ISO 8601, UTC+2, отчетени от сървъра (чл. 46).
