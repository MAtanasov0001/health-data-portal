# Terraform — скелет (режим B)

Само **скелет** с дефинирани граници. Целта е преносимост (cloud-agnostic): едни и същи модули
се прилагат в ДХЧО или друг съвместим облак.

**Извън обхвата на тази фаза (режим C — отложено):** мрежова сегментация, deny-by-default egress
филтриране, WAF, хранилище за тайни, жив SSRF-устойчив proxy, идентичност/автентикация. Тези
ресурси се добавят само след осигуряване на ревюиращи по режим C и външен тест за проникване.

## Структура

- `variables.tf` — входни променливи (без стойности по подразбиране, съдържащи тайни).
- `main.tf` — заготовка (locals + изисквания към версиите). Реалните ресурси се добавят по-късно.

Не съхранявайте състояние (state) с тайни в хранилището. Отдалеченото състояние и тайните се
конфигурират на Фаза 0.
