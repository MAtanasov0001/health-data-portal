# Метаданен профил — DCAT-AP 3.0.1 (здравен профил на портала)

Този каталог съдържа **машинно четимия метаданен профил** на портала. Профилът е приложение
на [DCAT-AP 3.0.1](https://semiceu.github.io/DCAT-AP/releases/3.0.1/) с уточнения за
статистически данни (StatDCAT-AP) и националните изисквания на data.egov.bg.

Целта е **интероперативност**: наборите от портала да се харвестват без загуба на смисъл в
[data.egov.bg](https://data.egov.bg) (CKAN) и [data.europa.eu](https://data.europa.eu), и да
отговарят на изискванията за отворени данни (машинна четимост, контролирани речници, версии).

## Съдържание

| Файл | Предназначение |
|------|----------------|
| [`profile.json`](profile.json) | Машинно четим профил: класове, свойства, задължителност, речници |
| [`vocabularies.md`](vocabularies.md) | Използваните контролирани речници и как се прилагат |
| [`examples/dataset.jsonld`](examples/dataset.jsonld) | Примерен набор в JSON-LD (валиден спрямо профила) |

## Нива на задължителност

Следваме конвенцията на DCAT-AP:

- **M** (mandatory) — задължително; без него наборът не се публикува.
- **R** (recommended) — препоръчително; използвай, когато е налично.
- **O** (optional) — по избор.

## Ключови класове

- **`dcat:Catalog`** — самият портал като каталог (един на инсталация).
- **`dcat:Dataset`** — набор от данни (концептуалната единица, която потребителят търси).
- **`dcat:Distribution`** — конкретен файл/достъп (CSV, JSON, XLSX, RDF) на набора.
- **`dcat:DataService`** — публичното API като услуга за достъп (чл. 41 — API-first).

## Национални уточнения

- **Времеви стойности** по ISO 8601, часова зона **UTC+2 (Europe/Sofia)**, генерирани от
  сървъра (чл. 46). Полетата `dct:issued` и `dct:modified` спазват това.
- **Уникален адрес (URI)** за всеки набор и distribution — достъпни чрез GET (МЕ72).
- **Версия** на набора (`owl:versionInfo`) — снапшотите са неизменяеми и версионирани;
  предходни версии остават достъпни (чл. 14 — 24 мес. поддръжка).
- **Лиценз** по подразбиране за данните: препоръка **CC BY 4.0** (фиксира се с governance).

## Статистически набори (StatDCAT-AP)

Здравните набори са агрегатна статистика (измерения + мярка), затова профилът добавя класа
**`StatisticalDataset`** ([StatDCAT-AP](https://semiceu.github.io/StatDCAT-AP/), namespace
`stat:`). Набор се разпознава като статистически по наличие на свойство с префикс `stat:`:

- **`stat:dimension`** (M) — измеренията на наблюдението (напр. област, диагностична група);
- **`stat:attribute`** (R) — мерната колона/единици (напр. брой хоспитализации);
- **`stat:numSeries`** (R) — брой серии/наблюдения в набора.

Тези свойства се генерират автоматично при приемане от спецификацията за контрол на
разкриването (измерения + мярка).

## Валидиране

Профилът се валидира в CI (порта „Python“) чрез **блокираща порта**: всеки JSON-LD пример в
`examples/` трябва да е валиден спрямо `profile.json`. Статистическите набори (носещи `stat:`
свойство) се проверяват и по задължителните StatDCAT-AP свойства. Пълната SHACL валидация
остава отделна стъпка при генериране на каталога. Локално:

```bash
# от ingestion/ — валидира примерите спрямо profile.json (DCAT-AP + StatDCAT-AP)
python -m ingestion.dcat.validate spec/dcat-ap/examples/*.jsonld
```
