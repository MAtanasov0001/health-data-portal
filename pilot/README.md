# Пилотен набор — Диагноза България (файлово подаване)

Този каталог съдържа **пилотния набор от данни** за портала, подаден по **файловия път**
(CSV + метаданни), без да се чака API от титуляр. Целта е тръбата да е осезаема end-to-end:

```
файл (CSV + метаданни) → схемна валидация → контрол на разкриването → неизменяем версиониран
снапшот → каталог (DCAT-AP) → публично API → frontend
```

## Съдържание

| Файл | Предназначение |
|------|----------------|
| [`hospitalizacii-po-oblast-2023/source.csv`](hospitalizacii-po-oblast-2023/source.csv) | Суров агрегиран вход (примерни/синтетични данни) |
| [`hospitalizacii-po-oblast-2023/metadata.yaml`](hospitalizacii-po-oblast-2023/metadata.yaml) | Метаданни на набора (влизат в DCAT-AP профила) |
| [`hospitalizacii-po-oblast-2023/schema.yaml`](hospitalizacii-po-oblast-2023/schema.yaml) | Схема на колоните за валидация при приемане |

## Важно за данните

Стойностите тук са **синтетични/примерни** и служат само за демонстрация на тръбата. Те **не
са реални здравни данни**. Реалните набори идват от титулярите вече агрегирани; порталът добавя
**независим контрол на разкриването** (small-cell suppression) върху агрегатите.

Стойности под прага за минимален размер на клетка се маркират в суровия вход като `<5`, за да
се провери, че приемната тръба ги разпознава и потиска коректно (без да позволява възстановяване
чрез редови/колонни суми).

## Стартиране на пилота през тръбата

```bash
cd ingestion
pip install -e ".[dev]"
python -m ingestion.pipeline run ../pilot/hospitalizacii-po-oblast-2023
# → създава версиониран снапшот в ingestion/snapshots/ с контролна сума и времеви печат (UTC+2)
```
